# 2026-05-01 — Radar completo Claude Code & OpenAI Codex

## Resumen ejecutivo

Hoy el mapa de agentic coding se está consolidando alrededor de dos líneas claras:

1. **Claude Code** está empujando fuerte hacia una plataforma completa: CLI, IDEs, desktop, web, mobile/remote-control, Agent SDK, plugins, skills, hooks, subagents, MCP, computer use, routines/scheduled tasks y revisiones cloud como `/ultrareview`.
2. **OpenAI Codex** acaba de publicar `0.128.0` como release estable reciente, con `0.129.0-alpha.2` ya disponible hoy en npm/GitHub. El foco nuevo de Codex está en `/goal` workflows persistentes, app-server APIs, perfiles explícitos de permisos/sandbox, plugins, hooks, imports de agentes externos, MultiAgentV2 y memoria/instrucciones ad-hoc.
3. **La práctica recomendada cambia:** ya no basta con “abrir un agente y pedirle código”. Lo más productivo es diseñar workflows: plan -> ejecución aislada -> review -> merge, con reglas versionadas (`CLAUDE.md`, `AGENTS.md`, skills), hooks de validación, permisos explícitos, MCP limitado y worktrees paralelos.

## Hallazgos clave de Claude Code

### Plataforma y superficies

Claude Code ya no es solo terminal. Las fuentes oficiales listan: terminal CLI, VS Code, JetBrains, desktop app, web, Chrome extension beta, computer use preview, Slack, GitHub Actions, GitLab CI/CD, Bedrock, Vertex AI, Microsoft Foundry, remote control, scheduled tasks/routines y Agent SDK.

**Insight:** si el objetivo es productividad real, conviene elegir la superficie por tipo de trabajo:

- CLI: refactors locales, debugging, tareas con repo grande.
- Desktop/web: planificación, ejecución aislada, sesiones paralelas, PRs cloud.
- IDE: diffs inline y navegación de contexto.
- SDK: construir agentes internos y automatizaciones programáticas.
- Slack/GitHub/GitLab: delegación desde flujos de equipo.

### Skills y comandos convergen

La documentación actual dice que **custom commands se han fusionado con skills**. Un `.claude/commands/deploy.md` y un `.claude/skills/deploy/SKILL.md` pueden crear `/deploy`; las skills agregan frontmatter, supporting files, control de invocación, carga bajo demanda, subagent execution y dynamic context injection.

**Tip:** convierte comandos repetidos en skills cuando tengan playbook, checklist, archivos de soporte o lógica multi-paso. Mantén `CLAUDE.md` para hechos estables; skills para procedimientos.

### Bundled skills y comandos útiles

Docs actuales mencionan bundled skills como `/simplify`, `/batch`, `/debug`, `/loop` y `/claude-api`. Además, el mapa de docs destaca `/ultrareview`, `/autofix-pr`, `/team-onboarding`, `/usage`, `/context`, `/hooks`, `/mcp`, `/agents`, `/compact`, `/plan`, `/review` y `/security-review`.

**Workflow recomendado:**

1. `/context` antes de tareas largas.
2. `/plan` o `/ultraplan` para cambios grandes.
3. Ejecutar en worktree o sesión aislada.
4. `/review`, `/security-review` o `/ultrareview` antes de merge.
5. `/compact` al 70–85% de contexto.

### Hooks mucho más potentes

La referencia actual de hooks incluye eventos mucho más amplios que el patrón clásico PreToolUse/PostToolUse/Stop. Aparecen eventos como `SessionStart`, `Setup`, `InstructionsLoaded`, `UserPromptSubmit`, `UserPromptExpansion`, `PreToolUse`, `PermissionRequest`, `PostToolUse`, `PostToolUseFailure`, `PostToolBatch`, `PermissionDenied`, `Notification`, `SubagentStart`, `SubagentStop`, `TaskCreated`, `TaskCompleted`, `Stop`, `StopFailure`, `TeammateIdle`, `ConfigChange`, `CwdChanged`, `FileChanged`, `WorktreeCreate`, `WorktreeRemove`, `PreCompact`, `PostCompact`, `SessionEnd`, `Elicitation`, `ElicitationResult`.

**Tip accionable:** usa hooks para políticas determinísticas, no para “pedirle al modelo que se acuerde”. Ejemplos:

- `PreToolUse`: bloquear `rm -rf`, `git push --force`, acceso a `.env`.
- `PostToolUse`: correr formatter/linter tras editar archivos.
- `SessionStart`: cargar estado de git, rama, ticket actual.
- `PreCompact`: guardar resumen o snapshot.
- `FileChanged`: ejecutar tests focalizados.

### Agent SDK como framework

El `llms.txt` oficial de Claude Code lista Agent SDK con Python/TypeScript, custom tools, MCP, permissions, plugins, skills, subagents, structured outputs, streaming input/output, sessions, checkpointing, OpenTelemetry y hosting.

**Framework mental:** Claude Code sirve como librería para construir agentes internos con las mismas capacidades del CLI: tools, permisos, memoria/proyecto, skills y observabilidad.

## Tips & tricks accionables de hoy

1. **Claude: latest vs stable.** `latest` es 2.1.126 y `stable` 2.1.118. Para exploración usa latest; para equipo productivo usa stable o pin explícito.
2. **Claude: skills para procedimientos.** Si un bloque en `CLAUDE.md` parece checklist o playbook, muévelo a skill para carga bajo demanda y menor coste de contexto.
3. **Claude: hooks como guardrails.** Implementa bloqueo determinístico en `PreToolUse` y validación en `PostToolUse` en vez de confiar en prompts de seguridad.
4. **Codex: migra de `--full-auto` a perfiles.** El release 0.128.0 depreca `--full-auto` y empuja permission profiles explícitos.
5. **Codex: prueba `/goal`, pero con cautela.** Es la novedad más fuerte, pero hay issues abiertos de TUI. Úsalo primero en repos no críticos.
6. **Codex: aprovecha `codex update`.** Ahora hay comando oficial de actualización en CLI; valida antes con version pin si trabajas en equipos.
7. **Ambos: versiona instrucciones.** Claude: `CLAUDE.md`, `.claude/skills`, hooks. Codex: `AGENTS.md`, config/requirements, skills/hooks. No dejes el “saber hacer” solo en chats.
8. **Ambos: worktrees paralelos.** Usa un worktree por issue/PR para evitar conflictos y facilitar review/rollback.
9. **Ambos: revisiones multi-agente.** Ejecuta implementación y review en sesiones separadas; no le pidas al mismo agente que sea juez único de su propio cambio.
10. **Ambos: observa contexto/coste.** Claude tiene `/context`, `/usage`, `/compact`; Codex está mejorando memoria, goals y tracing. Diseña checkpoints.

## Workflows recomendados

### Workflow 1 — Feature mediana con Claude Code

1. `claude -p "Analiza el repo y propón plan" --max-turns 5`
2. Revisar plan humano.
3. Sesión interactiva o worktree: `/plan`, implementar.
4. Hooks ejecutan formatter/tests.
5. `/review` o `/security-review`.
6. PR con resumen y riesgos.

### Workflow 2 — Issue paralelo con Codex

1. Crear worktree por issue.
2. `codex exec` o Codex App con permisos explícitos.
3. Usar `/goal` si la tarea es multi-día o persistente.
4. Revisar diffs y tests.
5. Evitar `--full-auto`; preferir perfiles/sandbox explícitos.

### Workflow 3 — Base reusable de equipo

1. Crear `CLAUDE.md` y `AGENTS.md` consistentes.
2. Extraer procedimientos a skills: deploy, migration, security-review, performance-audit.
3. Añadir hooks de seguridad y calidad.
4. Configurar MCP mínimo necesario por proyecto.
5. Crear “golden workflows” documentados: bugfix, feature, PR review, incident response.

## Frameworks / conceptos a seguir

- **Claude Agent SDK:** para construir agentes internos con tools, MCP, permissions, hooks, skills, subagents, sessions y OpenTelemetry.
- **Codex app-server:** para embeber Codex en productos o flujos propios.
- **MCP:** capa común de herramientas externas; cuidar límites de output, schemas y permisos.
- **Skills / Agent Skills:** estándar emergente para empaquetar procedimientos reusables; Claude y Codex están convergiendo en esta dirección.
- **Hooks:** automatización determinística alrededor del agente.
- **Worktrees:** aislamiento git como primitive básica para agentes paralelos.
- **Memory/instructions:** Claude con CLAUDE.md/auto memory; Codex con AGENTS.md/memories/ad-hoc instructions.
