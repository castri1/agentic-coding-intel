Ecosistema

Skills + hooks + MCP como stack común

Ambas herramientas convergen en instrucciones versionadas, skills reutilizables, MCP, hooks y subagents. La ventaja está en diseñar workflows repetibles y auditables.

  • Versiona CLAUDE.md / AGENTS.md.
  • Usa hooks para seguridad y calidad.
  • Un worktree por agente/tarea.

Top tips de hoy

  • Pin de canal: Claude latest para exploración, stable para equipos; Codex alpha solo pruebas.
  • Procedimientos a skills: deploys, migrations, reviews, incident response y performance audits.
  • Permisos explícitos: Codex 0.128 empuja permission profiles; evita depender de --full-auto.
  • Hooks como políticas: bloquear comandos peligrosos, correr formatters/tests, capturar contexto antes de compactar.
  • Review independiente: que un agente implemente y otro revise. Evita juez único.
  • Worktrees paralelos: una tarea por worktree; merge/PR al final.

Gotchas actuales

  • Codex 0.128 tiene reportes abiertos sobre /goal, Shift+Enter macOS, sandbox background commands, MCP schema conversion y hooks de image generation.
  • Claude Code tiene muchos canales/superficies: verifica si el comando aplica a CLI, desktop, web o SDK.
  • MCP aumenta poder y riesgo: limita output, scopes y permisos por servidor.
  • Skills no deben ser dumps: frontmatter + pasos claros + archivos de soporte.
